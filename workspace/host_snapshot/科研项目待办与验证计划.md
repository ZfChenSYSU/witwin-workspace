# 科研项目待办与验证计划

版本：2026-07-07

项目题目：一种使用手机采集 CSI 并优化室内无线可微射线追踪的新方法

## 1. 当前研究定位

本项目的目标是验证一种低成本室内无线数字孪生构建方案：使用手机采集视频和 IMU 数据，使用成熟 SLAM 系统输出手机轨迹，同时使用 PicoScenes 支持的 Wi-Fi 网卡或 SDR 采集手机空口通信帧对应的 CSI，并将这些数据用于优化室内无线可微射线追踪模型。

根据当前讨论，本项目暂时采用以下取舍：

1. **不再将 SLAM + IMU 轨迹滤波作为主线贡献。**

   轨迹直接使用成熟 SLAM 系统输出。项目只需要验证 SLAM 轨迹是否能与 CSI 做时间同步和坐标对齐，不需要额外设计轨迹滤波算法。
2. **重点验证人体-手机相对距离误差 `w_{hd}`。**

   核心问题是：真实采集中人体与手机之间的距离和姿态并不固定，而仿真中人体模型通常设置为固定距离，例如 40 cm。这个差异是否会显著影响 CSI 与可微射线追踪仿真结果的匹配，是本项目最值得验证的科学问题之一。
3. **采用实验标定先验作为处理 `w_{hd}` 的优先路线。**

   相比直接从 CSI 中反推人体-手机距离，先通过实验标定得到 `d_t` 的统计范围、变化速度和先验分布，更稳妥，也更容易形成可验证的实验闭环。
4. **`w_{hd}` 的科学假设验证应先于完整硬件闭环。**

   在投入较多硬件调试之前，应该先在仿真条件下验证人体-手机相对距离扰动是否会显著影响 CSI，以及这种影响是否足以导致环境参数 `\theta` 的错误更新。如果仿真中 `w_{hd}` 的影响很弱或不可辨识，后续就不应把复杂的 `d_t` 估计作为主线贡献。
5. **硬件可行性仍需尽早并行确认。**

   硬件不再作为唯一的第一阶段主线，但仍需尽早确认当前网卡、路由器和手机通信方式是否能够稳定读取有效 CSI，并确认读取到的 CSI 究竟对应哪一条无线链路。尤其需要区分“手机到路由器的真实链路”和“手机到旁路监听网卡的可观测链路”。
6. **根据 PicoScenes 文档，原始硬件方案需要修正表述。**

   旁路网卡监听手机与路由器通信是可行方向，但它得到的是“发射端到监听网卡”的 CSI，而不是路由器端实际接收到的 CSI。若继续使用旁路监听网卡，则无线数字孪生中的接收端应建模为监听网卡；若必须研究手机到路由器的真实信道，则需要路由器/AP 端本身具备 CSI 输出能力，或使用支持 PicoScenes 的设备替代普通路由器作为接收端。

## 2. 核心待验证假设

### 2.1 硬件假设

- [ ] H-HW-1：现有网卡或路由器能够读取手机通信过程中的 CSI。
- [ ] H-HW-2：读取到的 CSI 具有稳定时间戳、子载波信息、幅度和相位信息。
- [ ] H-HW-3：CSI 能够与手机位置轨迹进行时间同步。
- [ ] H-HW-4：读取到的 CSI 确实对应预期信道，例如手机到路由器，或路由器到手机，而不是手机到旁路监听网卡。
- [ ] H-HW-5：若使用旁路监听网卡，则实验模型明确改写为“手机到监听网卡”的 CSI 测量，而不是“手机到路由器”的 CSI 测量。
- [ ] H-HW-6：若必须获得路由器端 CSI，则路由器/AP 端设备必须本身支持 CSI 输出，普通家用路由器通常不能默认满足该条件。

其中 H-HW-4 和 H-HW-5 是关键风险点。PicoScenes 支持 AX210/AX200 等网卡在 monitor mode 下对 overheard frames 做 CSI measurement；这说明旁路监听手机上行帧是可行的。但该 CSI 的接收端是监听设备，而不是该数据包原本的目标路由器。若如此，系统模型需要调整：要么将监听网卡视为接收端并纳入无线数字孪生，要么改用支持 CSI 输出的 AP/路由器作为接收端。

### 2.2 科学假设

- [ ] H-SCI-1：人体-手机相对距离变化 `d_t` 会对测量 CSI 产生可观测影响。
- [ ] H-SCI-2：`d_t` 的变化在自然手持过程中是有界且相对平滑的。
- [ ] H-SCI-3：使用实验标定得到的 `d_t` 先验，可以降低仿真 CSI 与测量 CSI 之间的残差。
- [ ] H-SCI-4：如果将 `d_t` 作为人体隐变量纳入优化，可减少环境参数 `\theta` 对人体误差的错误拟合。
- [ ] H-SCI-5：幅度-相位分离损失、鲁棒损失和异常点降权能够提升可微射线追踪优化稳定性。

### 2.3 工程假设

- [ ] H-ENG-1：手机视频和成熟 SLAM 系统可以输出足够稳定的手机轨迹。
- [ ] H-ENG-2：SLAM 坐标系能够与房间坐标系、AP 坐标系对齐。
- [ ] H-ENG-3：CSI、SLAM 轨迹和实验标定数据可以在统一时间轴上对齐。
- [ ] H-ENG-4：数据采集流程可以重复执行，并形成可复现实验数据集。

## 3. 总体技术路线

推荐将项目分为四条并行但有优先级的主线。当前应先做 `w_{hd}` 的仿真科学假设验证，同时以较低成本并行确认硬件路线，避免在尚未证明科学问题成立前投入过多硬件调试。

```text
主线 A：w_hd 仿真科学假设验证
    -> 构建简化室内仿真环境
    -> 人为扰动人体-手机距离 d_t
    -> 验证 w_hd 对 CSI 残差和 theta 偏差的影响
    -> 判断 d_t 建模是否值得作为主线贡献

主线 B：硬件 CSI 可行性验证
    -> 确认能否读取有效 CSI
    -> 确认 CSI 对应的物理链路
    -> 建立基础数据采集脚本

主线 C：人体-手机距离先验标定
    -> 外部测量 d_t
    -> 得到 d_t 的范围、均值、方差、变化速度
    -> 建立 w_hd 的先验模型

主线 D：真实数据闭环实验
    -> 同步采集 CSI、SLAM 轨迹、d_t 标定数据
    -> 计算测量-仿真残差
    -> 验证 w_hd 建模是否改善优化结果
```

优先级排序：

```text
w_hd 仿真科学假设验证
    > 硬件 CSI 最小可行性确认
    > 数据时间同步
    > d_t 标定实验
    > 仿真算法增强与真实参数先验接入
    > 真实数据下的可微射线追踪优化
```

## 4. 阶段一：`w_{hd}` 仿真科学假设验证

这是当前最应该优先推进的任务。原因是：如果在理想仿真条件下，人体-手机相对距离扰动 `d_t` 对 CSI 的影响都不显著，或者它对环境参数 `\theta` 的估计偏差很小，那么后续就没有必要把 `d_t` 估计、卡尔曼滤波或复杂人体先验作为项目主线。

该阶段不依赖真实硬件 CSI，可以先用简化射线追踪、几何光学近似或已有可微射线追踪工具完成。当前推荐优先评估 WiTwin，尤其是 `witwin-channel`，作为 `w_{hd}` 敏感性和路径级 CIR/CFR 仿真的候选工具。它的目标不是做出最终高保真仿真，而是回答一个科学问题：

```text
w_hd 是否足够显著，显著到值得被单独建模？
```

### 4.1 最小仿真场景

建议先构建一个尽量简单但可解释的室内环境：

- [ ] 一个 AP 或监听网卡接收端。
- [ ] 一个手机发射端轨迹 `loc_t`。
- [ ] 一个简化房间，例如四面墙、地面、天花板。
- [ ] 一个简化人体模型，例如圆柱体、椭球体或长方体。
- [ ] 固定频率，例如 2.4 GHz；后续再扩展到 5 GHz。
- [ ] 固定环境参数真值 `\theta^*`，例如墙面反射系数、介电常数。
- [ ] 设置人体-手机默认距离 `d_0`，例如 40 cm。
- [ ] 若使用 WiTwin，先跑通 `witwin-channel` 的最小 path solver 示例，再加入人体模型。

初始仿真不必追求复杂家具和高阶多径。建议优先包含：

```text
直射路径
一次墙面反射
人体遮挡路径
人体反射/散射近似路径
```

### 4.2 `d_t` 扰动设计

需要人为生成几类人体-手机距离序列：

| 序列 | 设置                                      | 目的                               |
| ---- | ----------------------------------------- | ---------------------------------- |
| D-0  | 固定 `d_t = d_0`                        | 无距离扰动 baseline                |
| D-1  | 小幅随机扰动，例如 `d_t = d_0 +/- 2 cm` | 测试轻微手持抖动                   |
| D-2  | 中等扰动，例如 `d_t = d_0 +/- 10 cm`    | 模拟自然手持变化                   |
| D-3  | 大幅扰动，例如 `d_t = d_0 +/- 20 cm`    | 测试极端姿态变化                   |
| D-4  | 缓慢周期变化                              | 模拟手臂自然摆动                   |
| D-5  | 突然跳变                                  | 模拟换手、调整握持或稳定器姿态突变 |

每组都应记录真实 `d_t`，用于后续评价估计误差。

### 4.3 第一组验证：`w_{hd}` 是否改变 CSI

固定环境参数 `\theta^*` 和手机位置 `loc_t`，只改变人体-手机距离 `d_t`，观察 CSI 变化。

待办：

- [ ] 计算不同 `d_t` 下的复数 CSI。
- [ ] 分别分析幅度变化和相位变化。
- [ ] 比较直射路径未遮挡、直射路径被人体遮挡、人体近场反射增强三种情形。
- [ ] 画出 `d_t` 与 CSI 幅度残差的关系。
- [ ] 画出 `d_t` 与 CSI 相位残差的关系。
- [ ] 找到 `d_t` 影响明显和不明显的空间位置。

判断标准：

```text
如果 d_t 改变 5-10 cm 就能造成稳定可观测的幅度/相位变化，
则 w_hd 值得继续建模。

如果只有极端 d_t 变化才有影响，
则可以将 w_hd 降级为鲁棒损失或域随机化问题。
```

### 4.4 第二组验证：错误固定 `d_0` 是否导致 `\theta` 偏差

生成仿真观测数据时使用真实扰动 `d_t`，但优化时错误地固定人体距离为 `d_0`，观察环境参数 `\theta` 是否被错误更新。

对照组：

| 组别  | 生成数据      | 优化模型              | 目的                       |
| ----- | ------------- | --------------------- | -------------------------- |
| SIM-A | `d_t = d_0` | 固定 `d_0`          | 理想 baseline              |
| SIM-B | `d_t` 扰动  | 固定 `d_0`          | 观察忽略 `w_{hd}` 的偏差 |
| SIM-C | `d_t` 扰动  | 使用 `d_t` 真实值   | 理论上限                   |
| SIM-D | `d_t` 扰动  | 使用 `d_t` 先验分布 | 验证实验标定先验是否有价值 |
| SIM-E | `d_t` 扰动  | 离线估计 `d_t`      | 验证 `d_t` 是否可辨识    |

评价指标：

```math
E_s = \frac{1}{T}\sum_t ||s_t - \hat{s}_t||
```

```math
E_\theta = ||\theta - \theta^*||
```

```math
E_d = \frac{1}{T}\sum_t |d_t - \hat{d}_t|
```

关键判断：

- [ ] 若 SIM-B 的 `E_\theta` 明显大于 SIM-A，说明 `w_{hd}` 会污染环境参数估计。
- [ ] 若 SIM-D 明显优于 SIM-B，说明实验标定先验有价值。
- [ ] 若 SIM-E 接近 SIM-C，说明逐时刻估计 `d_t` 可能成为贡献点。
- [ ] 若 SIM-E 不稳定或不可辨识，则不建议优先推进 EKF/UKF。

### 4.5 第三组验证：`d_t` 是否可辨识

该验证直接决定是否值得做卡尔曼滤波或隐变量估计。

待办：

- [ ] 固定 `\theta = \theta^*`，只估计 `d_t`。
- [ ] 固定错误 `\theta`，再估计 `d_t`，观察是否混淆。
- [ ] 同时估计 `d_t` 和少量 `\theta`，观察是否病态。
- [ ] 测试不同噪声水平下 `d_t` 的恢复精度。
- [ ] 测试不同手机轨迹下 `d_t` 的恢复精度。

通过标准：

```text
在合理噪声水平下，d_t 平均误差能低于 5-10 cm，
且不同初始化结果稳定，才值得继续做 d_t 在线估计。
```

如果达不到该标准，建议只使用：

```text
d_t 统计先验 + 鲁棒损失 + 异常点降权
```

而不是强行做卡尔曼滤波。

这是需要尽早并行推进的任务。若无法稳定读取 CSI，后续真实实验需要调整数据来源或实验设备；但它不应阻塞阶段一的仿真科学假设验证。

### 5.2 原方案可行性判断

原始设想是：

```text
手机 <---- Wi-Fi ----> 路由器/AP
             |
             | 旁路网卡监听通信并读取 CSI
```

该设想需要拆成两个不同物理含义的方案。

#### 方案 A：旁路监听手机上行帧，测量“手机到监听网卡”的 CSI

这是当前最可行的低成本路线。

拓扑为：

```text
手机  --上行 Wi-Fi 帧-->  路由器/AP
  \
   \--同一上行帧被 AX210/AX200 monitor mode 监听-->
      监听网卡输出 CSI
```

此时 CSI 的物理含义是：

```text
phone -> monitoring NIC
```

而不是：

```text
phone -> router/AP
```

可行性判断：

- [X] PicoScenes 文档明确支持 AX210/AX200 在 monitor mode 下对 overheard frames 做 CSI measurement。
- [X] 手机无需安装特殊软件，只需要产生稳定上行流量。
- [X] 普通路由器可以继续作为网络接入点和流量目标。
- [ ] 必须在模型中把监听网卡作为接收端，而不是把路由器作为接收端。
- [ ] 若监听网卡与路由器位置不同，二者对应的射线追踪结果不可直接混用。

适合本项目的改写版本：

```text
使用手机移动采集视频/IMU/SLAM；
使用 AX210/AX200 监听手机上行 Wi-Fi 帧；
获得 phone -> monitoring NIC 的 CSI；
在可微射线追踪中将 monitoring NIC 的天线位置作为接收端位置。
```

#### 方案 B：获得“手机到路由器/AP”的真实路由器端 CSI

该方案不是普通旁路监听能直接完成的。

若要获得：

```text
phone -> router/AP
```

的路由器端 CSI，需要满足至少一个条件：

- 路由器/AP 本身支持 CSI 输出。
- 使用支持 PicoScenes 的设备作为接收端/AP。
- 使用改造过的 AP 或特殊 CSI 工具。
- 使用 SDR/USRP 作为接收端并将其纳入系统模型。

可行性判断：

- [ ] 普通家用路由器通常不能直接输出 CSI。
- [ ] 外置监听网卡即使能听到手机上行帧，输出的也是监听网卡接收该帧时的 CSI。
- [ ] 如果把监听网卡放在路由器旁边，只能近似 phone -> router/AP 信道；对相位敏感的优化来说，这个近似可能不够严格。

因此，原方案“通过旁路网卡监听路由器端得到 CSI”表述不严谨，需要改成：

```text
通过 AX210/AX200 旁路监听手机上行帧，获得手机到监听网卡的 CSI；
如需路由器端 CSI，则需更换为支持 CSI 输出的 AP/接收端。
```

### 5.3 建议的硬件路线

#### 路线 1：AX210/AX211 旁路监听路线

这是当前主线。

硬件：

- AX210 或 AX211 网卡。
- x86-64 Ubuntu 22.04 真机。
- 普通路由器/AP。
- 手机。

实验链路：

```text
手机上行 UDP/ping/iperf 流量
    -> 普通路由器/AP
    -> AX210 monitor mode 同信道监听
    -> PicoScenes 输出 phone -> AX210 的 CSI
```

优点：

- 成本低。
- 与手机和普通路由器兼容性较好。
- 支持 802.11ax 和较大带宽。
- 适合快速验证 `w_{hd}`。

缺点：

- CSI 接收端不是路由器。
- 需要精确记录 AX210 天线位置。
- 若使用相位信息，需要额外做相位校准和稳定性测试。

#### 路线 2：AX210/AX200 多网卡主动收发路线

如果希望更可控，可以使用一块 AX210/AX200 发包，另一块 AX210/AX200 收包并测 CSI。

优点：

- 发包速率、频段、带宽更可控。
- 不依赖手机 Wi-Fi 协议栈产生稳定帧。
- 便于先验证算法。

缺点：

- 手机不再是无线发射端，偏离原始手机采集设定。
- 更适合作为硬件和算法 baseline，不适合作为最终主实验。

#### 路线 3：支持 CSI 输出的 AP/接收端路线

如果论文必须声称测量的是“手机到 AP/路由器”的 CSI，则需要推进该路线。

可能方式：

- 使用支持 CSI 输出的 Linux AP 或接收端。
- 使用改造过的 QCA9300 AP 或类似平台。
- 使用 USRP/SDR 作为接收端。

优点：

- 物理链路定义最接近原始问题。

缺点：

- 成本和工程复杂度显著上升。
- 普通家用路由器大概率不可用。
- 可能偏离低成本路线。

| HW-3     | 手机缓慢移动                | 检查 CSI 是否随位置变化连续变化                            |
| HW-4     | 人体遮挡直射路径            | 检查 CSI 幅度是否出现明显衰减                              |

| HW-6     | 只分析手机上行帧            | 验证 phone -> AX210/AX200 链路是否随手机位置变化           |
| HW-7     | 只分析 AP 下行帧            | 验证 router/AP -> AX210/AX200 链路，避免与手机上行链路混淆 |

### 5.7 判断 CSI 是否有效

最小有效标准：

- [ ] 能输出每个数据包的时间戳。
- [ ] 能输出每个数据包的 RSSI 或类似强度指标。
- [ ] 能输出每个子载波的复数 CSI，至少包括实部和虚部，或幅度和相位。
- [ ] 能区分发射端和接收端 MAC。
- [ ] 能确定该 CSI 对应的无线链路。
- [ ] 分析手机上行帧时，手机移动会引起 CSI 幅度和相位随位置变化而变化。
- [ ] 分析 AP 下行帧时，手机移动不应被误解释为发射端位置变化，除非人体/手机作为环境扰动影响 AP -> 监听网卡链路。
- [ ] 人体遮挡时，CSI 出现可解释变化。

需要警惕的无效情况：

- [ ] 只能抓到 802.11 包，但没有 CSI。
- [ ] 只能得到 RSSI，不能得到子载波 CSI。
- [ ] CSI 时间戳不稳定或无法与其他数据对齐。
- [ ] 抓到的是旁路监听链路，而不是目标路由器链路。
- [ ] 相位完全不可用，且没有校准方法。
- [ ] 混合了手机上行帧和 AP 下行帧，导致 CSI 的发射端位置含义不一致。

## 7. 阶段四：人体-手机距离 `w_{hd}` 标定先验

该阶段是本项目的重点之一。目标不是一开始就精确估计每个时刻的 `d_t`，而是先回答以下问题：

1. 自然手持时，人体与手机距离大概是多少？
2. 这个距离的变化范围有多大？
3. 这个距离变化是否足以显著影响 CSI？
4. 用标定得到的距离先验是否能改善仿真-测量匹配？

### 7.1 定义人体-手机距离

需要先定义 `d_t`，否则后续标定会不一致。

推荐定义：

```text
d_t = 手机 Wi-Fi 天线中心 到 人体胸腔近似反射中心 的距离
```

如果手机天线中心难以确定，可以先近似为手机几何中心。

待办：

- [ ] 确认手机天线大致位置。
- [ ] 定义人体参考点，例如胸腔中心、躯干表面最近点、人体模型中心。
- [ ] 定义默认仿真距离 `d_0`，例如 40 cm。
- [ ] 定义合理距离范围，例如 `[25 cm, 70 cm]`，后续由实验修正。

### 7.2 标定方案 A：侧拍视频标定

这是成本最低、最推荐的起步方案。

实验布置：

```text
侧向相机
    |
    |  拍摄人体 + 手机
    |
实验人员手持手机/稳定器
```

待办：

- [ ] 在手机或稳定器上贴明显标记点。
- [ ] 在人体胸前或衣服上贴参考标记点。
- [ ] 侧向相机固定位置拍摄。
- [ ] 使用标定尺或已知长度物体换算像素到实际距离。
- [ ] 采集静态手持、慢速移动、自然移动三类数据。
- [ ] 从视频中提取手机标记点和人体标记点距离。

优点：

- 实现简单。
- 不依赖复杂硬件。
- 能直观看到手持姿态变化。

缺点：

- 只能测侧向投影距离。
- 如果人体或手机不在同一平面，会有投影误差。

适合作为初始标定，不适合作为最终精确测量。

### 7.3 标定方案 B：深度相机标定

如果有深度相机，可以更稳定地测量人体与手机三维距离。

待办：

- [ ] 固定深度相机位置。
- [ ] 标定深度相机坐标系与房间坐标系。
- [ ] 识别手机或稳定器上的标记点。
- [ ] 识别人体躯干区域或胸腔参考点。
- [ ] 输出 `d_t` 时间序列。
- [ ] 与侧拍视频结果交叉验证。

优点：

- 可以得到三维距离。
- 标定数据质量更高。

缺点：

- 设备要求更高。
- 深度相机可能受遮挡、反光、衣物材质影响。

### 7.4 标定方案 C：人工分档测量

这是最粗糙但最容易完成的方案，适合快速验证。

设置几个固定手持距离：

```text
d = 25 cm, 35 cm, 45 cm, 55 cm, 65 cm
```

待办：

- [ ] 使用尺子或固定支架控制手机到人体距离。
- [ ] 每个距离采集 1-3 分钟 CSI。
- [ ] 保持手机位置和 AP 位置不变。
- [ ] 只改变人体-手机距离。
- [ ] 分析 CSI 幅度和相位随 `d` 的变化。

这个实验非常重要，因为它直接验证 `d_t` 是否真的影响 CSI。如果在静态可控条件下，不同 `d` 对 CSI 的影响都不明显，那么后续复杂估计 `d_t` 的价值就会降低。

### 7.5 标定数据分析

标定结束后，需要得到以下统计量：

- [ ] `d_t` 的均值 `d_0`。
- [ ] `d_t` 的标准差 `\sigma_d`。
- [ ] `d_t` 的最小值 `d_min` 和最大值 `d_max`。
- [ ] `d_t` 的变化速度分布 `\dot{d}_t`。
- [ ] 不同手持方式下的 `d_t` 分布。
- [ ] 不同实验人员之间的 `d_t` 差异。

可以建立先验：

```math
d_t \sim \mathcal{N}(d_0, \sigma_d^2)
```

并加入硬约束：

```math
d_{min} \le d_t \le d_{max}
```

还可以建立动态先验：

```math
d_{t+1} = d_t + v_t \Delta t + q_t
```

其中：

```math
q_t \sim \mathcal{N}(0, \sigma_q^2)
```

- [ ] `human_distance_calibration.md`：记录标定方法。
- [ ] `human_distance_prior.json` 或 `yaml`：保存先验参数。
- [ ] `human_distance.csv`：保存每次实验的 `d_t` 时间序列。
- [ ] `d_distribution.png`：距离分布图。
- [ ] `d_velocity_distribution.png`：距离变化速度分布图。
- [ ] `csi_vs_d.png`：CSI 特征随距离变化的图。

阶段四通过标准：

- 得到可重复的 `d_t` 统计范围。
- 能回答不同 `d` 是否显著影响 CSI。
- 能给出用于后续仿真和优化的 `d_t` 先验。

### 8.2 仿真对照组

| 组别  | 设置                                 | 目的                                       |
| ----- | ------------------------------------ | ------------------------------------------ |
| SIM-0 | 无人体扰动，固定 `d_0`             | 验证基础射线追踪优化是否正常               |
| SIM-1 | 有人体扰动，但算法仍使用固定 `d_0` | 观察 `w_{hd}` 对残差和 `\theta` 的影响 |
| SIM-2 | 使用标定先验，但不估计逐时刻 `d_t` | 验证先验是否能提高鲁棒性                   |
| SIM-3 | 将 `d_t` 作为隐变量优化            | 验证是否能恢复距离扰动                     |
| SIM-4 | 使用错误的 `d_t` 先验              | 测试算法对错误先验的敏感性                 |

### 8.3 推荐损失函数

对于复数 CSI，建议使用幅度-相位分离损失：

```math
L_{csi} =
\lambda_A \rho_A(A_t - \hat{A}_t)
+ \lambda_\phi \rho_\phi(wrap(\phi_t - \hat{\phi}_t))
```

加入人体距离先验：

```math
L =
\sum_t \omega_t L_{csi}(t)
+ \lambda_d \sum_t (d_t - d_0)^2
+ \lambda_s \sum_t (d_t - d_{t-1})^2
+ \lambda_\theta R(\theta)
```

其中：

- `\omega_t`：异常点权重。
- `\rho_A`、`\rho_\phi`：Huber 或 Cauchy 鲁棒损失。
- `\lambda_d`：距离先验强度。
- `\lambda_s`：距离时间平滑强度。
- `R(\theta)`：环境参数正则项。

### 8.4 `d_t` 的估计方法

建议分三个层级实现。

#### Level 1：只使用静态先验

不估计每个时刻的 `d_t`，只在仿真中使用标定得到的均值和方差。

做法：

- 固定 `d = d_0`。
- 或从 `d_t \sim N(d_0, \sigma_d^2)` 中采样多个人体距离，做域随机化。
- 优化 `\theta` 时对多个 `d` 的仿真结果求平均或取鲁棒损失。

优点：

- 实现最简单。
- 不容易出现不可辨识问题。

缺点：

- 无法解释逐时刻距离变化。

#### Level 2：离线平滑估计 `d_t`

将 `d_t` 作为整段轨迹上的待优化变量，通过先验和时间平滑约束估计。

做法：

```math
\{d_t\}_{t=1}^T =
\arg\min_{d_t}
\sum_t L_{csi}(s_t, F_\theta(loc_t, d_t))
+ \lambda_d \sum_t (d_t - d_0)^2
+ \lambda_s \sum_t (d_t - d_{t-1})^2
```

优点：

- 比在线卡尔曼滤波更容易调试。
- 可以作为验证 `d_t` 是否可辨识的第一步。
- 与卡尔曼平滑在思想上接近。

缺点：

- 不是实时算法。
- 需要可微射线追踪能够对 `d_t` 求梯度，或能进行网格搜索。

#### Level 3：EKF/UKF 估计 `d_t`

当 Level 2 证明 `d_t` 可辨识后，再实现滤波器。

状态定义：

```math
X_t =
\begin{bmatrix}
d_t \\
\dot{d}_t
\end{bmatrix}
```

状态转移：

```math
d_{t+1} = d_t + \dot{d}_t \Delta t + q_d
```

```math
\dot{d}_{t+1} = \dot{d}_t + q_v
```

观测模型：

```math
s_t = F_\theta(loc_t, h = 1, d_t) + \epsilon_t
```

由于 `F_\theta` 是非线性的，应使用 EKF 或 UKF。

优点：

- 形式清晰，适合写成算法贡献。
- 可以在线估计 `d_t`。

难点：

- 观测是高维 CSI，滤波器可能不稳定。
- `d_t` 与 `\theta`、人体介电参数之间可能混淆。
- 需要先有较可靠的 `\theta` 或交替优化策略。

推荐先做 Level 1 和 Level 2，再决定是否推进 Level 3。

### 8.5 仿真评价指标

- [ ] `CSI residual`：测量 CSI 与仿真 CSI 的残差。
- [ ] `theta error`：估计环境参数与真值的误差。
- [ ] `d_t error`：估计人体距离与真值的误差。
- [ ] `generalization error`：在未参与优化的轨迹上评估残差。
- [ ] `stability`：不同初始化下是否得到相近结果。

推荐指标：

```math
E_\theta = ||\theta - \theta^*||
```

```math
E_d = \frac{1}{T}\sum_t |d_t - \hat{d}_t|
```

```math
E_s = \frac{1}{T}\sum_t ||s_t - \hat{s}_t||
```

阶段五通过标准：

- 固定错误距离 `d_0` 时，残差明显上升。
- 使用标定先验或估计 `d_t` 后，残差下降。
- `\theta` 的估计偏差小于不处理 `w_{hd}` 的 baseline。

## 12. 风险清单与应对策略

### 12.1 风险：硬件无法读取 CSI

表现：

- 只能抓包，不能输出 CSI。
- 网卡或路由器芯片不受支持。
- CSI 工具不兼容当前系统。

应对：

- [ ] 优先更换为 PicoScenes 明确支持的 AX210/AX211 或 AX200/AX201。
- [ ] 确认 Ubuntu 22.04 真机、Secure Boot 关闭、网卡接口可用。
- [ ] 改用能输出 CSI 的路由器平台。
- [ ] 将项目初期转为仿真验证和标定先验验证。
- [ ] 降级使用 RSSI 做极简验证，但不能替代最终 CSI 实验。

### 12.2 风险：读取到的 CSI 链路不是目标链路

表现：

- 外置网卡监听到的是手机到监听网卡的信道。
- 但模型想优化的是手机到路由器的信道。

应对：

- [ ] 若采用 AX210/AX200 monitor mode，明确将监听网卡作为接收端，重定义系统模型为 phone -> monitoring NIC。
- [ ] 将监听网卡尽量放置在 AP 附近时，仍需承认二者位置不同；尤其是相位建模中，厘米级天线偏移都可能显著改变相位。
- [ ] 改用 AP 端 CSI 输出。
- [ ] 在论文中清楚定义 CSI 链路，不混用“路由器端 CSI”和“监听端 CSI”。

### 12.3 风险：混合上行帧和下行帧导致建模对象混乱

表现：

- 手机上行帧对应 `phone -> monitoring NIC`。
- AP 下行帧对应 `AP/router -> monitoring NIC`。
- 若两类帧混在一起，`loc_t` 与 CSI 发射端位置不再一致。

应对：

- [ ] 使用 MAC 地址和帧方向筛选手机上行帧。
- [ ] 单独保存上行数据集和下行数据集。
- [ ] 主线优化只使用手机上行帧。
- [ ] AP 下行帧只作为环境扰动或稳定性对照，不直接用于手机位置相关的射线追踪优化。

### 12.4 风险：`d_t` 对 CSI 影响不可辨识

表现：

- 不同人体-手机距离下 CSI 差异不稳定。
- 差异被环境多径、人体材质、姿态变化淹没。

应对：

- [ ] 先做静态分档距离实验。
- [ ] 如果分档实验无明显差异，降低 `d_t` 估计优先级。
- [ ] 改为只使用 `d_t` 统计先验和域随机化。
- [ ] 将重点转向鲁棒损失和异常降权。

### 12.5 风险：相位不可用

表现：

- 相位存在随机跳变。
- 相位受硬件频偏、采样偏移影响严重。

应对：

- [ ] 初期只使用幅度验证。
- [ ] 后续再做相位校准和 unwrap。
- [ ] 在损失函数中降低相位权重。
- [ ] 将相位作为辅助指标，而不是第一阶段主指标。

### 12.6 风险：真实环境不可控

表现：

- 人员移动、家具变化、其他 Wi-Fi 干扰影响 CSI。

应对：

- [ ] 使用独立房间或尽量安静的实验时段。
- [ ] 固定 AP 信道。
- [ ] 记录环境变化。
- [ ] 每轮实验前后采集短基准数据。

## 15. 建议的近期执行顺序

建议按以下顺序推进，不要一开始就直接做完整算法：

```text
1. 先在仿真中证明人体-手机距离 d_t 的变化确实会影响 CSI
2. 再验证错误固定 d_0 是否会导致 theta 的估计偏差
3. 再判断 d_t 是否可辨识，决定是否值得做隐变量估计或 EKF/UKF
4. 并行做 AX210/AX200/PicoScenes 的最小硬件可行性确认
5. 再分离手机上行帧和 AP 下行帧，确认主线 CSI 链路为 phone -> monitoring NIC
6. 再建立 d_t 的实验标定先验
7. 最后把先验和真实 CSI 放进可微射线追踪优化
```

原因是：`w_{hd}` 是否值得复杂建模，首先取决于它在理想仿真中是否显著、是否会污染 `\theta`、以及 `d_t` 是否可辨识。硬件 CSI 当然必须尽早确认，但不应成为科学假设验证的前置阻塞项。

## 16. PicoScenes 资料来源与当前结论

本节记录本次根据 PicoScenes 官方文档得到的硬件和方案判断，后续硬件采购或实验设计应优先回查这些页面。

参考资料：

- PicoScenes 首页与总览：https://ps.zpj.io/
- Supported Hardware：https://ps.zpj.io/manual/hardware.html
- CSI Measurement using PicoScenes：https://ps.zpj.io/manual/scenarios.html
- PicoScenes Installation & Upgrade：https://ps.zpj.io/manual/installation.html

当前结论：

1. PicoScenes 明确支持的主要 CSI 硬件包括 AX210/AX211、AX200/AX201、QCA9300、IWL5300、HackRF One 和 USRP 系列。
2. 对本项目而言，AX210/AX211 是首选，AX200/AX201 是 2.4/5 GHz 下的备选。
3. AX210/AX200 支持 monitor mode 下 overheard frames 的 CSI measurement，因此旁路监听手机上行帧是可行路线。
4. 旁路监听得到的 CSI 是发射端到监听网卡的信道，不是普通路由器端接收到的信道。
5. 原方案若继续使用普通路由器和旁路网卡，应改写为 `phone -> monitoring NIC` 的 CSI 测量方案。
6. 如果必须获得 `phone -> router/AP` 的真实路由器端 CSI，则需要 AP/路由器本身支持 CSI 输出，或使用 PicoScenes 支持设备替代普通路由器作为接收端。
